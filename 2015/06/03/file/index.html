访问文件

iOS应用程序只能在自己的目录下进行文件操作，不可以访问其他的存储空间，此区域被称为沙盒。在应用开发中所有的非代码文件都要保存在此，例如图片、声音、各种文档、文本文件等。

Library目录

这个目录下有两个子目录：Caches和Preferences

Library/Preferences目录，包含应用程序的偏好设置文件。不应该直接创建偏好设置文件，而是应该使用NSUserDefaults类来取得和设置应用程序的偏好。

Library/Caches目录，主要存放缓存文件，iTunes不会备份此目录，此目录下文件不会在应用程序退出时删除。

NSFileHandle

NSFileHandle是对文件描述的面向对象封装，可以通过他的句柄对象去访问文件、套接字、管道、设备等信息。前面NSFileManager我们可以实现对文件及目录进行管理、而通过NSFileHandle，我们可以更深层次的对文件内的数据进行管理

NSBundle

获取NSBundle里的资源

我们通常使用类方法NSBundle.mainBundle来获取当前应用可执行文件的目录位置。对于一个app，他的mainBundle通常指向.app这个跟目录。
使用bundleURL属性和bundlePath属性可以查询bundle路径。

如果希望获得bundle目录下的文件，可以直接通过文件名以及扩展名称
