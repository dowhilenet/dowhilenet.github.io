UIScrollView 继承UIView ：UIResponder : NSObject

子类有: UICollectionView UITableView UITextView

主要的作用的就是可以显示多于一个屏幕的内容或者超过你能放在内存中的内容

我们来说说他的一些属性：

Managing the Display of Content

contentOffset属性

是内容视图坐标原点与ScrollView坐标原点的偏移量，返回CGPoint结构体类型，这个结构体类型包含X和Y两个成员。

偏移量也可以通过ScrollView的类方法来设定，带有动画效果

func setContentOffset(_ contentOffset: CGPoint,animated animated: Bool)

contentSize

表示ScrollView中内容视图的大小，它返回CGSize结构体类型，该结构体包含了width和height两个成员。

contentInset

用于ScrollView的内容周围添加边框。这往往为了留出空白以放置工具栏、标签栏或者导航栏。

contentInset属性有四个分量，分别是Top、Left、Right、Bottom。

Managing Scrolling

ScrollEnabled

默认是true，如果是false的话Scrolling不能被激活，这不能接受触摸事件。

directionalLockEnabled

默认是False。如果是true的话，你往水平方向移动的话就会被被锁定在这个方向，不能进行竖直方向移动。

scrollsToTop

这个就是来控制点击了顶部的Status Bar之后返回到顶部。true OR false

scrollRectToVisible

pagingEnabled

当值是 YES 会自动滚动到 subview 的边界。默认是NO

bounces

默认是 yes，就是滚动超过边界会反弹有反弹回来的效果。假如是 NO，那么滚动到达边界会立刻停止。

alwaysBounceVertical || alwaysBounceHorizontal 和Bounces类似。

touchesShouldBegin

决定自己是否接收 touch 事件
Return false if you don’t want the scroll view to send event messages to view. If you want view to receive those messages, return true (the default).

touchesShouldCancelInContentView

开始发送 tracking messages 消息给 subview 的时候调用这个方法，决定是否发送 tracking messages 消息到subview。假如返回 NO，发送。YES 则不发送。

canCancelContentTouches

假如 canCancelContentTouches属是NO，则不调用touchesShouldCancelInContentView这个方法来影响如何处理滚动手势

delaysContentTouches

如果属性值是false则会立刻之行touchesShouldBegin方法，如果是true这会推迟

decelerationRate

设置手指放开后的减速率

tracking

当 touch 后还没有拖动的时候值是YES，否则NO

decelerating

当滚动后，手指放开但是还在继续滚动中。这个时候是 YES，其它时候是 NO

Managing the Scroll Indicator

indicatorStyle

设置样式

scrollIndicatorInsets

用于ScrollView的内容周围添加边框

showsHorizontalScrollIndicator

是否显示水平的滚动条

showsVerticalScrollIndicator

是否显示竖直方向的滚动条

flashScrollIndicators

显示滚动条，在你任何时候需要它都可以调用这类方法

UIScrollViewDelegate

Responding to Scrolling and Dragging

scrollViewDidScroll

只要滚动了就会触发

scrollViewWillBeginDragging

当开始滚动视图时，执行该方法。一次有效滑动（开始滑动，滑动一小段距离，只要手指不松开，只算一次滑动），只执行一次。

scrollViewWillEndDragging

滑动scrollView，并且手指离开时执行。一次有效滑动，只执行一次。 当pagingEnabled属性为YES时，不调用，该方法

scrollViewDidEndDragging

滑动视图，当手指离开屏幕那一霎那，调用该方法。一次有效滑动，只执行一次。

scrollViewShouldScrollToTop

如果你不是完全滚动到滚轴视图的顶部，你可以轻点状态栏，那个可视的滚轴视图会一直滚动到顶部，那是默认行为，你可以通过该方法返回NO来关闭它

scrollViewDidScrollToTop

到达顶部之后执行

scrollViewWillBeginDecelerating

滑动减速时调用该方法。

scrollViewDidEndDecelerating

减速停止了时执行，手触摸时执行执行

Managing Zooming

viewForZoomingInScrollView

返回将要缩放的UIView对象。要执行多次

scrollViewWillBeginZooming

当将要开始缩放时，执行该方法。一次有效缩放，就只执行一次。

scrollViewDidEndZooming

当缩放结束后，并且缩放大小回到minimumZoomScale与maximumZoomScale之间后（我们也许会超出缩放范围），调用该方法。

scrollViewDidZoom

当scrollView缩放时，调用该方法。在缩放过程中，回多次调用

Responding to scrolling animations

scrollViewDidEndScrollingAnimation

滚动动画停止时执行,代码改变时出发,也就是setContentOffset改变时
